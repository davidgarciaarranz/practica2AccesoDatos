Fecha: 2026-01-13
Proyecto: hetero3 (ruta raíz del workspace)

Resumen de cambios realizados:
- Archivo modificado: pom.xml
  - Sustituido:
    - Se eliminó `org.springdoc:springdoc-openapi-ui:1.7.0` (no compatible con Spring Boot 4).
  - Añadido/Actualizado:
    - `org.springdoc:springdoc-openapi-starter-webmvc-ui:2.1.0` (starter compatible con Spring Boot 4).
    - `io.swagger.core.v3:swagger-annotations:2.2.8` (anotaciones OpenAPI v3, opcional).
    - `org.springframework.boot:spring-boot-starter-hateoas` (para compatibilidad si se requiere HATEOAS).
  - Eliminado:
    - `org.springframework.boot:spring-boot-starter-data-rest` (no usado en el proyecto; evitó que SpringDoc intentara auto-configurar DataRest/HATEOAS).

- Archivo modificado: src/main/resources/application.properties
  - Añadida la exclusión de autoconfiguraciones de SpringDoc HATEOAS/DataRest para evitar errores al arrancar:
    spring.autoconfigure.exclude=org.springdoc.core.configuration.SpringDocHateoasConfiguration,org.springdoc.core.configuration.SpringDocDataRestConfiguration

Acciones realizadas (comprobaciones):
- Compilación/packaging:
  - `.\mvnw.cmd -DskipTests package` (build exitoso después de los cambios).
- Ejecución:
  - Arranqué el JAR: `java -jar target/hetero3-0.0.1-SNAPSHOT.jar` (logs redirigidos a `app.out.log` y `app.err.log` localmente).
- Verificaciones HTTP:
  - `GET http://localhost:8088/v3/api-docs` → Devuelve JSON OpenAPI con los endpoints del controlador (`/inicio/...`).
  - `GET http://localhost:8088/swagger-ui/index.html` → HTTP 200 (Swagger UI accesible).

Observaciones:
- Se detectó inicialmente un fallo de arranque por incompatibilidades de SpringDoc con las dependencias existentes (excepción relacionada con HATEOAS/DataRest). Se resolvió añadiendo la dependencia HATEOAS o excluyendo las autoconfiguraciones no deseadas.
